# Contracts Directory

Contracts are grouped by product area so protocol contributors, integrators, and reviewers can reason about the system by domain instead of by release accident.

- `exchange/`: AMM and swap infrastructure
- `payflow/`: intent execution and value routing
- `dao/`: governance and incentive infrastructure
- `agents/`: agent marketplace and execution rails
- `treasury/`: reward custody and escrow components
- `XpgnToken/`: token contract shared across protocol layers
- `mocks/`: non-production test support contracts
